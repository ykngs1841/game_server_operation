## 프로젝트 개요
게임 서버 운영 환경을 가정하여  
서버 실행, 상태 확인, 로그 관리, 자동화 구조를 학습 및 구현한 프로젝트

시스템 엔지니어 / DevOps 직무 관점에서
"서비스를 안정적으로 운영하기 위한 최소 구성"을 목표로 합니다.

## 주요 구성
- Linux (WSL2 / Ubuntu)
- Git 기반 형상 관리
- Docker 기반 서비스 실행
- 운영자 관점 헬스체크 및 로그 확인


## 디렉토리 구조
- game-server : 게임 서버(더미 서비스)
- docker : 컨테이너 실행 환경
- monitoring : 서버 상태 확인 스크립트 (추후 확장)


## 목표
- 게임 서버 운영 환경 이해
- 장애 발생 시 원인 파악 능력 강화
- 시스템 엔지니어 실무 감각 확보
